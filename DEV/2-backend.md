# 1. Что такое Backend

## 1.1 Определение простыми словами

**Backend** = серверная часть приложения, которую пользователь не видит

### Аналогия с рестораном

```
Frontend (зал ресторана)
↓
┌─────────────────────────┐
│ 👤 Посетитель           │ ← Видит меню, заказывает
│ 📋 Официант             │ ← Принимает заказ
└─────────────────────────┘

Backend (кухня ресторана)
↓
┌─────────────────────────┐
│ 👨‍🍳 Повар              │ ← Готовит блюдо
│ 📦 Холодильник          │ ← Хранит продукты
│ 📝 Рецепты              │ ← Логика приготовления
└─────────────────────────┘
```

**Посетитель не видит кухню**, но именно там происходит вся работа!

---

## 1.2 Frontend vs Backend

### Визуальное сравнение

```
╔═══════════════════════════════════╗
║         CutCode                   ║
╠═══════════════════════════════════╣
║  [Логин: _______]                 ║ ← FRONTEND
║  [Пароль: ______]                 ║   (видишь)
║  [Войти]                          ║
╚═══════════════════════════════════╝
          ↓ Отправка данных
┌───────────────────────────────────┐
│  Сервер:                          │
│  1. Проверить логин/пароль        │ ← BACKEND
│  2. Найти в базе данных           │   (не видишь)
│  3. Создать сессию                │
│  4. Отправить ответ               │
└───────────────────────────────────┘
```

### Таблица различий

| Критерий | Frontend | Backend |
|----------|----------|---------|
| **Где работает** | В браузере пользователя | На сервере компании |
| **Что видно** | ✅ Да, всё видно | ❌ Скрыто от пользователя |
| **Технологии** | HTML, CSS, JavaScript | Python, Node.js, PHP, Go |
| **За что отвечает** | Красивый интерфейс | Логика и данные |
| **Пример** | Кнопка "Купить" | Проверка наличия товара |

## 1.3 Примеры Backend в жизни

### Пример 1: Онлайн-магазин

```
Frontend:
Ты видишь товары, добавляешь в корзину, жмёшь "Купить"

Backend:
- Проверяет есть ли товар на складе
- Списывает деньги с карты
- Создаёт заказ в базе данных
- Отправляет уведомление на склад
- Отправляет email с подтверждением
```

### Пример 2: Социальная сеть

```
Frontend:
Ты пишешь пост, жмёшь "Опубликовать"

Backend:
- Сохраняет текст поста в базу данных
- Проверяет текст на запрещённые слова
- Отправляет уведомления подписчикам
- Обрабатывает лайки и комментарии
```

### Пример 3: Telegram бот

```
Frontend:
Пользователь пишет сообщение боту

Backend:
- Получает сообщение от Telegram API
- Обрабатывает текст (понимает что хочет пользователь)
- Ищет ответ в базе знаний
- Отправляет ответ обратно пользователю
```

**Главное:** Backend = всё что происходит "за кулисами"

# 2. Какие задачи решает Backend

## 2.1 Обработка данных

**Задача:** Принять данные от пользователя → обработать → вернуть результат

**Примеры:**

```
Калькулятор:
Frontend: пользователь вводит "2 + 2"
Backend: вычисляет результат → 4
Frontend: показывает "4"

Поиск:
Frontend: пользователь вводит "купить iPhone"
Backend: ищет в базе товаров → находит 50 вариантов
Frontend: показывает список товаров
```

---

## 2.2 Хранение данных

**Задача:** Сохранить информацию чтобы она не потерялась

**Что хранится:**
- 👤 Пользователи (логины, пароли, профили)
- 📝 Контент (посты, комментарии, файлы)
- 🛒 Транзакции (заказы, платежи)
- 📊 Статистика (кто когда заходил)

**Где хранится:** В базе данных (подробно в `DEV/3-base.md`)

---

## 2.3 Бизнес-логика

**Задача:** Реализовать правила работы приложения

**Примеры правил:**

```
Интернет-магазин:
- Скидка 10% если заказ > 5000₽
- Бесплатная доставка если заказ > 3000₽
- Можно вернуть товар в течение 14 дней

Социальная сеть:
- Можно опубликовать не больше 10 постов в час
- Видео можно загружать только подписчикам
- Нельзя отправлять больше 50 сообщений в день

Telegram бот:
- Отвечать только на разрешённые команды
- Записывать пользователя на консультацию
- Сохранять историю диалога
```

**Backend следит** чтобы правила соблюдались!

---

## 2.4 Безопасность

**Задача:** Защитить данные пользователей

**Что делает Backend:**

```
🔒 Хэширование паролей (чтобы безопасно их хранить в базе данных)
Пользователь вводит: "mypassword123"
Backend сохраняет: "$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy"

🔐 Проверка прав доступа
Пользователь пытается: Удалить чужой пост
Backend проверяет: Это не твой пост → Отказ!

🛡️ Защита от атак
Пользователь отправляет: 1000 запросов в секунду
Backend блокирует: Слишком много запросов → Бан на 1 час

✅ Валидация данных
Пользователь вводит: Email = "это не email"
Backend проверяет: Неправильный формат → Ошибка
```

**Без Backend-безопасности** любой мог бы украсть данные!

# 3. Из чего состоит Backend (обычно)

## 3.1 Сервер

**Сервер** = компьютер, на котором работает твой код 24/7

### Что это такое

```
Твой компьютер:
- Включаешь когда нужно
- Выключаешь когда спишь
- Только ты пользуешься

Сервер:
- Работает круглосуточно
- Никогда не выключается
- Готов к нагрузкам - тысячи людей одновременно подключаются
```

### Где находятся серверы

**Варианты:**

1. **Облачные серверы (shared)** (популярный выбор)
   - Timeweb
   - Beget

2. **VPS (Virtual Private Server)**
   - Арендуешь виртуальный сервер
   - Сам настраиваешь всё

3. **Свой физический сервер**
   - Покупаешь железо
   - Ставишь у себя в офисе
   - Редко используется сегодня

**Для воркшопа:** Будем использовать облачный VPS

---

## 3.2 База данных

База данных — это компонент backend, где хранятся данные приложения.

Подробный разбор (SQL, NoSQL, CRUD, примеры) вынесен в DEV/3-base.md.
---

## 3.3 Код приложения

**Код приложения** = программа на сервере, которая обрабатывает запросы

### Что делает код

```python
# Псевдокод (не настоящий Python)

# 1. Принять запрос
запрос = получить_от_frontend()

# 2. Обработать
if запрос == "показать товары":
    товары = база_данных.получить_все_товары()
    
# 3. Отправить ответ
отправить_frontend(товары)
```

**Код решает:**
- Что делать с запросом
- Куда обращаться (к какой таблице БД)
- Что отправить в ответ

## 3.4 Как это работает вместе

### Полная схема Backend-системы

```
┌─────────────────────────────────────────────┐
│          ПОЛЬЗОВАТЕЛЬ (Frontend)            │
│  [Открывает сайт / Пишет боту]              │
└────────────────┬────────────────────────────┘
                 │
                 ↓ Отправляет запрос
┌─────────────────────────────────────────────┐
│              СЕРВЕР                         │
│                                             │
│  ┌──────────────────────────────────┐      │
│  │   КОД ПРИЛОЖЕНИЯ (Python)        │      │
│  │   - Получает запрос              │      │
│  │   - Обрабатывает логику          │      │
│  │   - Обращается к БД              │      │
│  └──────────┬───────────────────────┘      │
│             │                               │
│             ↓                               │
│  ┌──────────────────────────────────┐      │
│  │   БАЗА ДАННЫХ (PostgreSQL)       │      │
│  │   - Хранит пользователей         │      │
│  │   - Хранит товары/посты/данные   │      │
│  └──────────────────────────────────┘      │
└────────────────┬────────────────────────────┘
                 │
                 ↓ Возвращает ответ
┌─────────────────────────────────────────────┐
│          ПОЛЬЗОВАТЕЛЬ (Frontend)            │
│  [Видит результат]                          │
└─────────────────────────────────────────────┘
```

### Пример: Пользователь авторизуется

```
1. Frontend отправляет:
   { "email": "ivan@mail.ru", "password": "12345" }

2. Сервер получает запрос:
   - Код приложения проверяет формат данных
   - Ищет пользователя в БД по email
   - Сравнивает пароли

3. База данных возвращает:
   { "id": 1, "name": "Иван" }

4. Сервер отправляет Frontend:
   { "success": true, "user": { "id": 1, "name": "Иван" } }

5. Frontend показывает:
   "Добро пожаловать, Иван!"
```

**Вся магия** происходит на шагах 2-3!

# 6. Как Frontend общается с Backend

## 6.1 Запрос-ответ

### Базовая схема

```
┌──────────────┐                    ┌──────────────┐
│   Frontend   │                    │   Backend    │
│  (Браузер)   │                    │   (Сервер)   │
└──────┬───────┘                    └──────┬───────┘
       │                                   │
       │ 1. Запрос (Request)               │
       │ "Дай мне список товаров"          │
       ├──────────────────────────────────>│
       │                                   │
       │                      2. Обработка │
       │                      - Ищет в БД  │
       │                      - Формирует  │
       │                        ответ      │
       │                                   │
       │ 3. Ответ (Response)               │
       │ [товар1, товар2, товар3]          │
       │<──────────────────────────────────┤
       │                                   │
       │ 4. Отображение                    │
       │ Показывает товары пользователю    │
       │                                   │
```

### HTTP протокол

**HTTP** = протокол передачи данных в интернете

**Основные методы:**

```
GET    → Получить данные
POST   → Создать данные
PUT    → Обновить данные
DELETE → Удалить данные
```

---

## 6.2 Пример: Нажал кнопку → что произошло

### Сценарий: Пользователь регистрируется

**Шаг 1: Пользователь заполняет форму**

```html
<!-- Frontend -->
<form>
  <input name="name" value="Иван">
  <input name="email" value="ivan@mail.ru">
  <input name="password" value="12345">
  <button>Зарегистрироваться</button>
</form>
```

**Шаг 2: Frontend отправляет запрос**

```javascript
// JavaScript на Frontend
fetch('https://api.example.com/register', {
  method: 'POST',
  body: JSON.stringify({
    name: "Иван",
    email: "ivan@mail.ru",
    password: "12345"
  })
})
```

**Шаг 3: Backend получает и обрабатывает**

```python
# Python на Backend
@app.post('/register')
def register(data):
    # 1. Проверить данные
    if not validate_email(data.email):
        return {"error": "Неверный email"}
    
    # 2. Хэшировать пароль
    hashed = hash_password(data.password)
    
    # 3. Сохранить в БД
    user = database.save({
        "name": data.name,
        "email": data.email,
        "password": hashed
    })
    
    # 4. Отправить ответ
    return {"success": True, "user_id": user.id}
```

**Шаг 4: Frontend получает ответ**

```javascript
// JavaScript на Frontend
.then(response => {
  if (response.success) {
    alert("Регистрация успешна!");
    redirect('/profile');
  } else {
    alert("Ошибка: " + response.error);
  }
})
```

## 🎯 ИТОГОВОЕ РЕЗЮМЕ

### Что ты теперь знаешь:

**Backend - это:**
- ✅ Серверная часть приложения (невидимая для пользователя)
- ✅ Обработка данных и бизнес-логики
- ✅ Работа с базами данных
- ✅ Безопасность и авторизация

**Состав Backend-системы:**
- ✅ Сервер (где работает код)
- ✅ База данных (где хранятся данные)
- ✅ Код приложения (обрабатывает запросы)


**Языки программирования:**
- ✅ Python - простой для новичков
- ✅ Node.js - JavaScript на сервере
- ✅ PHP - для веб-сайтов

**Фреймворки:**
- ✅ Django/Flask/FastAPI (Python)
- ✅ Express (Node.js)
- ✅ aiogram (для Telegram ботов)

**Взаимодействие:**
- ✅ Frontend отправляет запрос
- ✅ Backend обрабатывает
- ✅ Frontend получает ответ
- ✅ JSON для передачи данных

### Главное правило vibe-coding backend:

```
Ты не должен быть профессиональным backend-разработчиком.
Ты должен понимать КАК работает backend.
Ты должен уметь проверить что сделал AI.
```
