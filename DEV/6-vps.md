## 2. Популярные VPS-провайдеры

### 2.1 Критерии выбора

При выборе VPS смотри на:
- **Цена** — от 200-500₽/месяц для старта
- **Оплата** — по часам или по месяцам, российские карты
- **Локация** — где физически находится сервер
- **Операционная система** — Ubuntu 22.04/24.04 (самая популярная)
- **Ресурсы** — CPU, RAM, диск
- **Техподдержка** — русскоязычная, скорость ответа

## 3. Создание VPS (на примере Timeweb)

### 3.1 Регистрация

1. Заходишь на [timeweb.com](https://timeweb.com)
2. Нажимаешь "Регистрация"
3. Вводишь email и пароль
4. Подтверждаешь email
5. Вводишь данные (ФИО, телефон)
6. Пополняешь баланс (минимум 100₽)

💳 **Способы оплаты:**
- Банковская карта (Мир, Visa, MasterCard)
- ЮMoney
- Qiwi
- WebMoney
- Криптовалюта

### 3.2 Создание VPS сервера

```
1. Заходишь в раздел "Облачные серверы" → "Создать сервер"

2. Выбираешь тариф:
   ✅ VPS-3 — 599₽/мес (2GB RAM, 2 CPU, 40GB SSD)

3. Выбираешь операционную систему:
   ✅ Ubuntu 24.04 LTS
   (или Ubuntu 22.04 LTS — тоже норм)

4. Выбираешь дата-центр:
   ✅ Москва (если живёшь в Европейской части РФ)
   ✅ С-Петербург (альтернатива)

5. Дополнительные опции:
   ☑ Автоматические бэкапы (рекомендуется, +50₽/мес)
   ☐ IPv6 (можно не включать)
   ☐ Защита от DDoS (не нужна для старта)

6. Способ доступа:
   ✅ SSH ключ (безопаснее, рекомендуется)
   ⚠️ Пароль (проще, но менее безопасно)

7. Имя сервера:
   Например: workshop-bot или my-landing-server

8. Нажимаешь "Создать сервер"
```

⏱️ **Создание займёт 1-3 минуты**

### 3.3 Получение доступа

После создания ты увидишь:

## 4. SSH — подключение к серверу

### 4.1 Что такое SSH

**SSH (Secure Shell)** — это защищённый способ подключиться к серверу и управлять им через терминал.

**Аналогия:**  
SSH = удалённый рабочий стол, но только в текстовом формате.

### 4.2 Подключение с разных систем

#### **macOS / Linux**

У тебя уже есть терминал. Открываешь и пишешь:

```bash
ssh root@185.123.45.67
```

(замени `185.123.45.67` на IP твоего сервера из панели Timeweb)

Если первое подключение:
```
The authenticity of host can't be established.
Are you sure you want to continue? (yes/no)
```
Пишешь: `yes`

Потом вводишь пароль (если настроил через пароль).

✅ **Ты подключён!**

---

#### **Windows**

**Способ 1: PowerShell** (Windows 10+)

PowerShell уже поддерживает SSH:
```powershell
ssh root@185.123.45.67
```

**Способ 2: PuTTY** (если нет SSH)

1. Скачиваешь [PuTTY](https://www.putty.org/)
2. В поле **Host Name** вводишь IP: `185.123.45.67`
3. Port: `22`
4. Connection type: SSH
5. Нажимаешь **Open**
6. Вводишь `root` как логин
7. Вводишь пароль

**Способ 3: Консоль в браузере Timeweb** (самый простой)

1. Открываешь панель управления Timeweb
2. Выбираешь свой сервер
3. Нажимаешь кнопку **"Консоль"**
4. Открывается терминал прямо в браузере
5. Вводишь логин `root` и пароль

✅ **Удобно для новичков — не нужно ничего устанавливать!**

---

### 4.3 SSH Key (рекомендуемый способ)

**SSH Key** — это пара ключей (приватный + публичный), которые заменяют пароль.

**Плюсы:**
- Безопаснее пароля
- Не нужно каждый раз вводить пароль
- Защита от брутфорса

#### Генерация SSH ключа

**macOS / Linux:**
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Нажимаешь Enter → Enter → Enter (сохранится в `~/.ssh/id_ed25519`)

**Windows PowerShell:**
```powershell
ssh-keygen -t ed25519 -C "your_email@example.com"
```

#### Копирование публичного ключа на сервер

**Через интерфейс Timeweb (проще всего):**

1. При создании сервера выбираешь "SSH ключ"
2. Вставляешь содержимое файла `~/.ssh/id_ed25519.pub`
3. Сервер создаётся уже с твоим ключом

**Вручную (если сервер уже создан):**

1. Смотришь публичный ключ:
```bash
cat ~/.ssh/id_ed25519.pub
```

2. Копируешь вывод (начинается с `ssh-ed25519 ...`)

3. Подключаешься к серверу по паролю:
```bash
ssh root@185.123.45.67
```

4. На сервере создаёшь файл:
```bash
mkdir -p ~/.ssh
nano ~/.ssh/authorized_keys
```

5. Вставляешь скопированный публичный ключ
6. Сохраняешь: `Ctrl+O` → Enter → `Ctrl+X`
7. Права доступа:
```bash
chmod 600 ~/.ssh/authorized_keys
chmod 700 ~/.ssh
```

Теперь можешь подключаться без пароля:
```bash
ssh root@185.123.45.67
```

## 5. Базовые команды Linux (Ubuntu)

После подключения ты в терминале сервера. Вот что тебе нужно знать.

### 5.1 Навигация

```bash
pwd
# Показывает текущую директорию
# /root

ls
# Список файлов и папок

ls -la
# Список с подробностями (размер, права, скрытые файлы)

cd /var/www
# Перейти в папку /var/www

cd ..
# Вернуться на уровень выше

cd ~
# Вернуться в домашнюю директорию (/root)
```

### 5.2 Работа с файлами

```bash
mkdir my-project
# Создать папку

touch index.html
# Создать пустой файл

nano index.html
# Открыть файл в редакторе nano
# Ctrl+O — сохранить
# Ctrl+X — выйти

cat index.html
# Показать содержимое файла

rm file.txt
# Удалить файл

rm -rf folder/
# Удалить папку со всем содержимым
# ⚠️ ОСТОРОЖНО! Без подтверждения!

cp file.txt file_copy.txt
# Скопировать файл

mv file.txt /var/www/
# Переместить файл
```

### 5.3 Установка программ

Ubuntu использует пакетный менеджер **apt**.

```bash
sudo apt update
# Обновить список доступных пакетов

sudo apt upgrade
# Обновить установленные пакеты

sudo apt install python3
# Установить Python 3

sudo apt install nginx
# Установить веб-сервер nginx

sudo apt install postgresql
# Установить PostgreSQL

```

💡 **sudo** = "выполнить от имени администратора"

### 5.4 Управление процессами

```bash
ps aux
# Список всех запущенных процессов

top
# Интерактивный монитор процессов (Ctrl+C чтобы выйти)

htop
# Красивый монитор (нужно установить: apt install htop)

kill 1234
# Остановить процесс с ID 1234

killall python3
# Остановить все процессы python3
```

### 5.5 Права доступа

```bash
chmod 755 script.sh
# Установить права (владелец: rwx, группа: rx, остальные: rx)

chmod +x script.sh
# Сделать файл исполняемым

chown user:group file.txt
# Сменить владельца файла
```

**Расшифровка прав:**
- `r` (read) — чтение
- `w` (write) — запись
- `x` (execute) — выполнение

```
755 = rwxr-xr-x
777 = rwxrwxrwx (все права для всех)
644 = rw-r--r-- (стандарт для файлов)
```

### 5.6 Поиск

```bash
find /var/www -name "*.html"
# Найти все .html файлы в /var/www

grep "error" /var/log/nginx/error.log
# Найти строки с "error" в файле логов

grep -r "TODO" .
# Рекурсивный поиск "TODO" в текущей папке
```

### 5.7 Архивы

```bash
tar -czvf archive.tar.gz folder/
# Создать архив folder → archive.tar.gz

tar -xzvf archive.tar.gz
# Распаковать archive.tar.gz

unzip file.zip
# Распаковать .zip
```

### 5.8 Сеть

```bash
curl http://example.com
# Сделать HTTP запрос

wget http://example.com/file.zip
# Скачать файл

ping google.com
# Проверить доступность сервера (Ctrl+C чтобы остановить)

ifconfig
# Показать сетевые интерфейсы (нужно: apt install net-tools)

ip addr
# Показать IP адреса (новая команда вместо ifconfig)
```

## 6. Первоначальная настройка сервера

### 6.1 Обновление системы

Сразу после создания VPS:

```bash
sudo apt update && sudo apt upgrade -y
```

Это обновит все пакеты до последних версий.

⏱️ Займёт 2-5 минут в зависимости от количества обновлений.

### 6.2 Создание нового пользователя (безопасность)

❌ **Плохо:** работать всегда под `root`  
✅ **Хорошо:** создать отдельного пользователя

```bash
# Создать пользователя (например, deploy)
adduser deploy

# Система попросит ввести пароль и информацию
# Вводишь пароль 2 раза, остальное можно пропустить (Enter)

# Добавить в группу sudo (чтобы мог использовать sudo)
usermod -aG sudo deploy

# Переключиться на нового пользователя
su - deploy
```

Теперь подключайся через:
```bash
ssh deploy@185.123.45.67
```

💡 **Почему это важно:**
- Если взломают `root` — доступ ко всему серверу
- Если взломают `deploy` — ограниченный доступ, меньше урона

### 6.3 Настройка файрвола (UFW)

**UFW (Uncomplicated Firewall)** — простой файрвол для Ubuntu.

```bash
# Включить UFW
sudo ufw enable

# Разрешить SSH (порт 22)
sudo ufw allow 22

# Разрешить HTTP (порт 80)
sudo ufw allow 80

# Разрешить HTTPS (порт 443)
sudo ufw allow 443

# Проверить статус
sudo ufw status
```

Вывод:
```
Status: active

To                         Action      From
--                         ------      ----
22                         ALLOW       Anywhere
80                         ALLOW       Anywhere
443                        ALLOW       Anywhere
```

⚠️ **ВНИМАНИЕ:** Не забудь разрешить порт 22 (SSH) ПЕРЕД включением UFW! Иначе потеряешь доступ к серверу.

### 6.4 Установка базовых инструментов

```bash
# Git
sudo apt install git -y

# Python и pip
sudo apt install python3 python3-pip -y

# PostgreSQL
sudo apt install postgresql postgresql-contrib -y

# nginx (веб-сервер)
sudo apt install nginx -y
```

⏱️ Установка всех пакетов займёт 5-10 минут.

### 6.5 Проверка установки

```bash
python3 --version
# Python 3.12.x

git --version
# git version 2.x.x

nginx -v
# nginx version: nginx/1.x.x

psql --version
# psql (PostgreSQL) 16.x
```

Если все команды выводят версии — **всё установлено правильно!** ✅

## 7. Как работать с сервером (workflow)

### 7.1 Типичный процесс деплоя

```
1. Разработка локально (на своём компьютере)
   ├─ Пишешь код
   └─ Тестируешь

2. Коммит в Git
   ├─ git add .
   ├─ git commit -m "Added landing page"
   └─ git push origin main

3. На сервере
   ├─ ssh deploy@server
   ├─ cd /var/www/my-project
   ├─ git pull origin main
   └─ Перезапуск сервисов (nginx, бот и т.д.)

4. Проверка
   └─ Открываешь сайт в браузере или проверяешь бота
```

### 7.2 Работа с логами

Логи — это текстовые файлы, где записывается что происходит.

```bash
# Логи nginx
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log

# Логи системы
sudo tail -f /var/log/syslog

# Логи PostgreSQL
sudo tail -f /var/log/postgresql/postgresql-16-main.log
```

💡 **tail -f** — показывает последние строки и обновляет в реальном времени (Ctrl+C чтобы выйти)

**Примеры что смотреть в логах:**

```bash
# Последние 50 строк логов nginx
sudo tail -50 /var/log/nginx/error.log

# Поиск ошибок
sudo grep "error" /var/log/nginx/error.log

# Логи за сегодня
sudo journalctl --since today
```

### 7.3 Автозапуск процессов (systemd)

Чтобы бот или сервер запускались автоматически после перезагрузки, используется **systemd**.

**Пример для Telegram бота:**

1. Создаёшь файл сервиса:
```bash
sudo nano /etc/systemd/system/telegram-bot.service
```

2. Содержимое:
```ini
[Unit]
Description=Telegram Bot
After=network.target

[Service]
Type=simple
User=deploy
WorkingDirectory=/var/www/telegram-bot
ExecStart=/usr/bin/python3 /var/www/telegram-bot/bot.py
Restart=always

[Install]
WantedBy=multi-user.target
```

3. Сохраняешь: `Ctrl+O` → Enter → `Ctrl+X`

4. Активируешь:
```bash
sudo systemctl daemon-reload
sudo systemctl enable telegram-bot
sudo systemctl start telegram-bot
```

5. Проверяешь статус:
```bash
sudo systemctl status telegram-bot
```

**Управление:**
```bash
sudo systemctl start telegram-bot    # Запустить
sudo systemctl stop telegram-bot     # Остановить
sudo systemctl restart telegram-bot  # Перезапустить
sudo systemctl status telegram-bot   # Статус
```

**Смотреть логи:**
```bash
sudo journalctl -u telegram-bot -f
```

## 8. Безопасность VPS

### 8.1 Основные правила

❌ **НИКОГДА не делай:**
- Работать всегда под `root`
- Хранить пароли в коде
- Открывать все порты
- Использовать слабые пароли
- Игнорировать обновления
- Отключать файрвол

✅ **ВСЕГДА делай:**
- Создавай отдельного пользователя
- Используй SSH ключи вместо паролей
- Настрой файрвол (UFW)
- Используй `.env` для секретов
- Обновляй систему регулярно
- Делай бэкапы

### 8.2 Отключение входа по паролю для root

После настройки SSH ключей:

```bash
sudo nano /etc/ssh/sshd_config
```

Найди и измени:
```
PermitRootLogin no
PasswordAuthentication no
```

Сохрани и перезапусти SSH:
```bash
sudo systemctl restart sshd
```

Теперь подключение только по SSH ключу для пользователя `deploy`.

### 8.3 Fail2Ban (защита от брутфорса)

**Fail2Ban** блокирует IP после нескольких неудачных попыток входа.

```bash
sudo apt install fail2ban -y

sudo systemctl enable fail2ban
sudo systemctl start fail2ban
```

Проверка:
```bash
sudo fail2ban-client status sshd
```

💡 Fail2Ban автоматически банит IP на 10 минут после 5 неудачных попыток входа.

### 8.4 Автоматические обновления

```bash
sudo apt install unattended-upgrades -y

sudo dpkg-reconfigure --priority=low unattended-upgrades
```

Выбери **Yes** — система будет автоматически устанавливать обновления безопасности.

### 8.5 Использование .env для секретов

❌ **ПЛОХО:**
```python
# bot.py
BOT_TOKEN = "123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11"
DATABASE_URL = "postgresql://user:password@localhost/db"
```

✅ **ХОРОШО:**
```python
# bot.py
import os
from dotenv import load_dotenv

load_dotenv()

BOT_TOKEN = os.getenv("BOT_TOKEN")
DATABASE_URL = os.getenv("DATABASE_URL")
```

```bash
# .env (не коммитить в Git!)
BOT_TOKEN=123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11
DATABASE_URL=postgresql://user:password@localhost/db
```

```bash
# .gitignore
.env
```

### 9.2 Очистка места

```bash
# Очистить кэш apt
sudo apt clean
sudo apt autoclean
sudo apt autoremove

# Найти большие файлы
sudo du -h --max-depth=1 /var/log | sort -h

# Очистить старые логи (оставить за последние 7 дней)
sudo journalctl --vacuum-time=7d
```

### 9.3 Бэкапы

❌ **Не полагайся только на сервер**  
✅ **Делай регулярные бэкапы**

**Способы:**

1. **Snapshots в панели Timeweb**
   - Заходишь в панель управления
   - Выбираешь сервер → "Снимки"
   - Создаёшь snapshot (занимает 2-5 минут)
   - Хранится 7 дней бесплатно

2. **Git для кода**
   - Весь код должен быть в Git (GitHub/GitLab)
   - `git push` после каждого изменения

3. **Экспорт БД PostgreSQL:**
```bash
# Создать дамп
pg_dump database_name > backup.sql

# Восстановить из дампа
psql database_name < backup.sql
```

💡 **Рекомендация:** делай бэкап перед каждым большим обновлением.
