# 1. Что такое база данных

**База данных** = структурированное хранилище информации

### Почему не просто файлы?

```
❌ ПЛОХО (файлы):
users.txt:
Иван,ivan@mail.ru,12345
Мария,maria@mail.ru,54321

Проблемы:
- Медленный поиск (нужно читать весь файл)
- Сложно обновлять (перезаписывать весь файл)
- Нет связей между данными
- Легко сломать формат

✅ ХОРОШО (база данных):
Таблица Users в PostgreSQL:
- Быстрый поиск (индексы)
- Легко обновлять (UPDATE запрос)
- Есть связи (Foreign Keys)
- Гарантия целостности данных
```

# 2. SQL базы данных (реляционные)

**SQL** = язык запросов для работы с таблицами

### Популярные SQL базы

- **PostgreSQL** — самая мощная и популярная
- **MySQL** — простая и быстрая
- **SQLite** — легкая (в одном файле)

### Как выглядят данные

```
Таблица "users":
┌────┬───────┬──────────────────┬──────────────┐
│ id │ name  │ email            │ created_at   │
├────┼───────┼──────────────────┼──────────────┤
│ 1  │ Иван  │ ivan@mail.ru     │ 2026-02-01   │
│ 2  │ Мария │ maria@mail.ru    │ 2026-02-02   │
└────┴───────┴──────────────────┴──────────────┘
```

**Связи между таблицами:**

```
Таблица "orders" ссылается на "users":
┌────┬─────────┬───────────┬────────┐
│ id │ user_id │ product   │ price  │
├────┼─────────┼───────────┼────────┤
│ 1  │ 1       │ iPhone    │ 80000  │ ← user_id = 1 = Иван
│ 2  │ 2       │ MacBook   │ 200000 │ ← user_id = 2 = Мария
└────┴─────────┴───────────┴────────┘
```

### Базовые SQL команды

```sql
-- Создать таблицу
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100)
);

-- Добавить данные
INSERT INTO users (name, email) 
VALUES ('Иван', 'ivan@mail.ru');

-- Получить данные
SELECT * FROM users;

-- Обновить данные
UPDATE users SET name = 'Иван Иванов' WHERE id = 1;

-- Удалить данные
DELETE FROM users WHERE id = 1;
```

**Это основы CRUD:**
- **C**reate (создать) = INSERT
- **R**ead (прочитать) = SELECT
- **U**pdate (обновить) = UPDATE
- **D**elete (удалить) = DELETE

# 3. NoSQL базы данных (нереляционные)

**NoSQL** = "Not Only SQL" = не только таблицы

### Популярные NoSQL базы

- **MongoDB** — хранит документы (JSON)
- **Redis** — хранит в памяти (очень быстро)
- **Elasticsearch** — для поиска

### Как выглядят данные в MongoDB

```json
// Коллекция "users" (вместо таблицы)
{
  "_id": "1",
  "name": "Иван",
  "email": "ivan@mail.ru",
  "address": {
    "city": "Москва",
    "street": "Ленина, 10"
  },
  "orders": [
    { "product": "iPhone", "price": 80000 },
    { "product": "AirPods", "price": 15000 }
  ]
}
```

**Особенности NoSQL:**
- Данные хранятся как документы (похоже на JSON)
- Нет жёстких связей между таблицами
- Легко добавлять новые поля
- Очень быстрые для чтения

# 4. Когда что использовать

### Сравнительная таблица

| Критерий | SQL (PostgreSQL) | NoSQL (MongoDB) |
|----------|------------------|-----------------|
| **Структура** | Жёсткая (таблицы, колонки) | Гибкая (JSON документы) |
| **Связи** | ✅ Сильные (Foreign Keys) | ⚠️ Слабые |
| **Изменение структуры** | Сложно (нужно менять схему) | Легко (просто добавил поле) |
| **Транзакции** | ✅ Надёжные | ⚠️ Ограниченные |
| **Скорость чтения** | Быстро | Очень быстро |
| **Где используется** | Банки, CRM, финтех | Соцсети, чаты, аналитика |

### Когда использовать SQL

```
✅ Используй SQL когда:
- Данные имеют чёткую структуру
- Много связей между таблицами
- Критична целостность данных
- Финансовые операции

Примеры:
- Интернет-магазин (пользователи + заказы + товары)
- CRM система
- Банковское приложение
- Бухгалтерия
```

### Когда использовать NoSQL

```
✅ Используй NoSQL когда:
- Данные не имеют строгой структуры
- Нужна высокая скорость
- Часто меняется формат данных
- Огромные объёмы данных

Примеры:
- Чат (сообщения)
- Соцсети (посты, лайки)
- Логи системы
- Кэш данных
```
